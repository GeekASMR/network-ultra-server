# Network Ultra Server

Center-relay audio server for [Network Ultra](https://github.com/GeekASMR/WDM2VST-Ultra) — a Windows VST3 plugin pair that lets two musicians collaborate over the network with **lossless FLAC** audio.

This is the open-source server side. The client plugins are closed-source and shipped with WDM2VST Ultra.

## Quick install (Linux)

SSH into your server and run:

```bash
curl -fsSL https://github.com/GeekASMR/network-ultra-server/releases/latest/download/install.sh | sudo bash
```

The script downloads the latest binary, generates a default config, registers a `systemd` service, and starts it. About 60 seconds end-to-end.

After install, point your VST plugin to `ws://<your-server-ip>:18900`.

## Manual install

1. Download the binary for your platform from [releases](https://github.com/GeekASMR/network-ultra-server/releases/latest):
   - `network-ultra-server-linux-amd64`
   - `network-ultra-server-linux-arm64`
   - `network-ultra-server-windows-amd64.exe` (experimental, no support)
2. Copy to `/usr/local/bin/network-ultra-server` and `chmod +x`
3. Create `/etc/network-ultra/config.toml` (see [config example](#config))
4. Run `network-ultra-server -config /etc/network-ultra/config.toml`

## Config

```toml
[server]
listen = "0.0.0.0:18900"
health_listen = "127.0.0.1:18901"
max_rooms = 50
max_peers_per_room = 8
max_connections = 200
admin_token = "<32-byte hex generated by install.sh>"

[tls]
enabled = false                  # set true to use cert_file/key_file or autocert below
cert_file = ""
key_file = ""
auto_letsencrypt = false
domain = ""
email = ""

[log]
level = "info"                   # debug | info | warn | error
format = "json"                  # json | text
path = ""                        # empty = stdout

[ratelimit]
hello_per_ip_per_minute = 10
room_create_per_peer_per_minute = 5
room_join_per_peer_per_minute = 30
room_list_per_peer_per_minute = 60
audio_frames_per_peer_per_second = 200
```

## Endpoints

- **WebSocket** `0.0.0.0:18900` — control + audio stream from VST clients
- **Health** `127.0.0.1:18901/healthz` — JSON status (uptime / rooms / peers)
- **Metrics** `127.0.0.1:18901/metrics` — Prometheus exposition

## TLS

Three modes via the `[tls]` section:

| Config | Mode | Use when |
|---|---|---|
| `enabled = false` | Plain `ws://` | LAN, friends-of-friends, simple self-host |
| `enabled = true` + `cert_file` + `key_file` | Static cert | You already have a cert |
| `enabled = true` + `auto_letsencrypt = true` + `domain` + `email` | autocert | Public-facing server with a domain |

When using autocert, ports 80 + 443 must be reachable from the internet (HTTP-01 challenge).

## Operations

```bash
systemctl status network-ultra-server      # is it alive?
journalctl -u network-ultra-server -f      # live logs (JSON by default)
systemctl restart network-ultra-server     # rolling restart (clients auto-reconnect)
curl http://127.0.0.1:18901/healthz        # quick health check
```
